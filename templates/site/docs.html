Introduction

Wisdom's service provides proxied API calls to various third party services. Eventually, these proxied API calls will carry identity and payment information, effectively implementing the 402 HTTP response code for proof of payment for a given call or credential.

For now, current the deployment supports adding new APIs, proxying calls to those APIs, and providing data about the calls being made.

Service APIs

Wisdom's service APIs may or may not require payment to make a call. Service APIs live at the following URL:

https://api.wisdom.sh/api/

Service Status

To get started with our APIs, issue a GET request to api.wisdom.sh for the /api/status endpoint in your bash terminal. A JSON formatted response giving system status is returned:

GET /api/status
curl -X GET https://api.wisdom.sh/api/status

{  
  "type": "GET",
  "status": "200",
  "view": "https://wisdom.sh/api/status",
  "timestamp": "2017-09-01 12:00:00",
  "qps": "2.0"
}

Authentication

Service endpoints on the /api/ path do not require authentication. However, some endpoints may require payment depending on demand for a given API resource. Endpoints on paths outside the /api/ service may or may not require authentication.

Add API

Wisdom proxies API endpoints which are added to the system by users. To add an API to the system, create a POST request, including the desired upstream_url to proxy and the desired uri to map onto the //api.wisdom.sh/ space.

POST /api/
curl -X POST \
  -d '{"upstream_url": "https://api.fixer.io/latest", "uri": "/eurorates"}' \
  https://api.wisdom.sh/api/

{  
  "type": "PUT",
  "status": "",
  "preserve_host":false,
  "created_at":1502942747239,
  "upstream_connect_timeout":60000,
  "upstream_url":"https:\/\/api.fixer.io\/latest",
  "uris":[  
    "\/eurorates"
   ],
  "view": "https://wisdom.sh/api/eurorates",
  "timestamp": "2017-09-01 12:00:00",
  "qps": "2.0"
}

Third Party APIs

Wisdom's API space includes proxied third party APIs. As an example, this GET request mapped to the /eurorates path returns a default JSON response from the /latest path on the fixer.io API:

GET //api.wisdom.sh/eurorates
curl https://api.wisdom.sh/eurorates

{  
   "base":"EUR",
   "date":"2017-09-01",
   "rates":{

}

API Status

Wisdom maps third party APIs and credentials for those APIs onto the //api.wisdom.com/ space. Use a GET request against the /api/ service path to query the status of the third party API and the proxy serving it:

GET /api/<api_id>/
curl https://api.wisdom.sh/api/eurorates/
