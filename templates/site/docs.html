Overview

{{ app_name }} is an instance of a Utter.io compute pool. Utter.io is like AirBnb for excess compute, providing fast location and provisioning of compute resources within a cooperative set of systems managed by OpenStack operators. Resource accounting inside the network is settled with Bitcoin and purchases of compute instances can be made by users without an account.

Providers can run a Utter.io virtual appliance on top of their OpenStack clusters, which allows a limited amount of control over the cluster by the appliance. Appliances listen to the Bitcoin blockchain for payments to a given Bitcoin address. When a payment is noted to an address, an instance is started using the flavor associated with the address.

The pool software provides a centralized location where appliances can advertise their instances for sale for a given ask price. By providing users the ability to bid on instances, this effectively creates a marketplace where compute can be openly sold, purchased or traded. These transactions are done via Instance Reservations:

Instances can only be started by payments made to an appliance's address. The appliance pool is not able to start a given appliance's instances directly, which should help eliminate payment fraud and general abuse of the system.

To elevate trust between users and providers, groups of operators can form adhoc hybrid clouds, which allows sharing of excess compute resources between trusted entities. Future version of the software will provide additional trust levels automatically.

Pool Accounts

Accounts on the system are provided via federated logins with Google. When you login to {{ app_name }} you will see a Google account chooser. Select the account you want to use on {{ app_name }}.

Once you've logged into your Google account, you will be informed about the level of access {{ app_name }} will have to your Google account. {{ app_name }} only needs access to your email address.
